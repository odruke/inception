# DEV_DOC

## Prerequisites
- Docker Engine and the Docker Compose plugin
- GNU Make
- OpenSSL (used to generate the self-signed certificate)

## Environment setup from scratch
1. Clone the repository and move to the repo root.
2. Create secrets files under `secrets/`:
   - `credentials.txt` with lines:
     - `DOMAIN_NAME <your_domain>`
     - `USER <your_wp_user>`
     - `ROOT <your_db_root_user>`
     - `USER_EMAIL <your_email>`
   - `db_password.txt` with the WordPress/DB user password
   - `db_root_password.txt` with the MariaDB root password
3. Configure the domain and storage paths as needed:
   - Domain is written to `DOMAIN_NAME` in `srcs/.env` and used by the Nginx template (`default.template`) and entrypoint script to generate the final `server_name` at container start.
   - Host data paths are defined by `MARIADB_VOL` and `WORDPRESS_VOL` in `Makefile` and also in the `device` entries under `volumes` in `srcs/docker-compose.yml`.
4. The `.env` file is generated automatically by `make build` and stored at `srcs/.env`.

## Build and launch
Makefile targets:
- `make build` - create host directories, generate SSL certs, write `.env`, build images, and start containers
- `make up` - start existing containers
- `make stop` - stop containers
- `make down` - stop and remove containers
- `make status` - show container status
- `make logs` - show recent nginx logs
- `make ssl` - generate SSL certificates if missing
- `make clean` - stop containers, remove `srcs/.env` and SSL certs, prune Docker artifacts
- `make re` - full rebuild (clean + build)

Docker Compose equivalents:
- `docker compose -f srcs/docker-compose.yml up --build`
- `docker compose -f srcs/docker-compose.yml up`
- `docker compose -f srcs/docker-compose.yml stop`
- `docker compose -f srcs/docker-compose.yml down`

## Manage containers and volumes
Common container commands:
- Status: `docker compose -f srcs/docker-compose.yml ps`
- Logs: `docker compose -f srcs/docker-compose.yml logs -f <service>`
- Shell: `docker compose -f srcs/docker-compose.yml exec <service> sh`
- Restart one service: `docker compose -f srcs/docker-compose.yml restart <service>`

Volume notes:
- Named volumes are backed by host bind mounts. `docker compose down -v` removes the Docker volume metadata, but the host directories still keep the data.
- To wipe data, remove or rename the host directories defined in the volume `device` paths.

## Data persistence
- MariaDB data persists in the host path mapped to `/var/lib/mysql`.
- WordPress files persist in the host path mapped to `/var/www/html`.
- Because these are bind mounts, data survives container rebuilds and `docker compose down`.

## SSL certificate
- The self-signed cert is generated by `srcs/tools/ssl.sh` into `srcs/requirements/nginx/ssl/` using `DOMAIN_NAME` from `secrets/credentials.txt`.
- If the domain changes, run `make clean` to remove the old certs, then `make build` to regenerate them.
